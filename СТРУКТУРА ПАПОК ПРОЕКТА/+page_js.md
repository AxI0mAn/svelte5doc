Что делать с +page.js. - его нужно дублировать для каждой страницы?

💡 Нужно ли дублировать +page.js?
Нет, файл +page.js не нужно дублировать для каждой страницы, если вы используете его только для настройки prerender: true.
1. ⚙️ Логика наследования
Опция export const prerender работает точно так же, как и макеты (+layout.svelte): она наследуется всеми дочерними маршрутами.
    • Если вы устанавливаете export const prerender = true; в корневой папке или в родительской группировке (как вы сделали в (home_all)/+page.js), то это значение автоматически применяется ко всем страницам, находящимся ниже в этой иерархии.
2. ✅ Анализ вашей структуры
Ваша текущая структура:
src/routes/
└── (home_all) 
    ├── +page.js  <-- Здесь export const prerender = true;
    ├── +page.svelte // Роут: /
    ├── home/
    │   └── settings/
    │          └── +page.svelte // Роут: /home/settings
    └── gameSet/
               └── +page.svelte // Роут: /gameSet
    • Благодаря тому, что вы поместили +page.js в папку (home_all), все роуты внутри этой группы (/, /home/settings, /gameSet) автоматически наследуют prerender: true.
    • Вам не нужно создавать отдельные файлы +page.js в папках settings/, gameSet/ или account/.

3. 📝 Когда +page.js нужен на каждой странице?
Файл +page.js становится необходим в конкретной папке маршрута только в двух случаях:
    1. Загрузка данных (load function): Если конкретная страница (например, /home/account) должна загружать свои уникальные данные, вы создадите +page.js с функцией export const load().
    2. Переопределение настроек: Если вам нужно отключить prerender для одной конкретной страницы внутри группы, вы создадите для нее +page.js и установите export const prerender = false;.
Вывод: Используйте +page.js только там, где необходимо либо настроить корневой роут, либо загрузить уникальные данные.
