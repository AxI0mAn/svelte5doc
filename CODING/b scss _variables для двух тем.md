Для приложения с двумя темами (например, Светлая и Темная) лучшая практика состоит в том, чтобы разделить переменные на три логических слоя внутри файла _variables.scss:
    1. Базовые/Структурные переменные: Не зависят от темы (шрифты, отступы, размеры).
    2. Цвета палитры: Не зависят от темы (чистые цвета).
    3. Семантические переменные: Зависят от темы (этот слой поглощает цвета палитры и дает им семантическое имя).
Вот пример кода и пояснения, как это должно выглядеть.

🎨 Пример файла _variables.scss для двух тем
SCSS
// =====================================================================
// _variables.scss
// Глобальные переменные: Макет + Цвета палитры + Семантические переменные
// =====================================================================

// ---------------------------------------------------------------------
// 1. СТРУКТУРНЫЕ ПЕРЕМЕННЫЕ (НЕ зависят от темы)
// ---------------------------------------------------------------------

// -- Типографика
$font-stack-base: 'Roboto', sans-serif;
$font-size-base: 1rem;
$font-weight-regular: 400;
$font-weight-bold: 700;

// -- Макет и отступы (Spacing)
$spacing-unit: 8px;
$spacing-md: $spacing-unit * 2; // 16px
$border-radius-base: 4px;
$transition-speed-base: 0.3s;

// ---------------------------------------------------------------------
// 2. ЦВЕТА БАЗОВОЙ ПАЛИТРЫ (НЕ зависят от темы)
// Чистые, "сырые" цвета.
// ---------------------------------------------------------------------

$palette-white: #ffffff;
$palette-black: #000000;
$palette-gray-light: #f5f5f5;
$palette-gray-dark: #333333;
$palette-blue: #2a7fec;
$palette-red: #dc3545;

// ---------------------------------------------------------------------
// 3. СЕМАНТИЧЕСКИЕ ПЕРЕМЕННЫЕ (Переключаемые)
// Эти переменные будут принимать разные значения в зависимости от темы.
// Мы устанавливаем здесь значения по умолчанию (например, для Светлой темы).
// ---------------------------------------------------------------------

// -- Фон и Основной текст
$color-bg-primary: $palette-gray-light;      // Фон страницы по умолчанию
$color-bg-secondary: $palette-white;         // Фон карточек/модальных окон
$color-text-primary: $palette-black;         // Основной цвет текста
$color-text-secondary: $palette-gray-dark;   // Второстепенный текст

// -- Интерактивные элементы
$color-accent: $palette-blue;               // Основной цвет бренда/кнопок
$color-border-default: $palette-gray-dark;
$color-link: $palette-blue;
$color-link-hover: $palette-blue;

// -- Статусы
$color-status-error: $palette-red;
$color-status-success: #28a745;

// ---------------------------------------------------------------------
// 4. МИКСИНЫ ТЕМЫ (Опционально: можно вынести в _mixins.scss)
// Миксины, которые переключают семантические переменные на другие значения.
// ---------------------------------------------------------------------

@mixin apply-theme-dark {
    // ⭐️ Переопределяем семантические переменные для темной темы
    $color-bg-primary: $palette-black !global;
    $color-bg-secondary: $palette-gray-dark !global;
    $color-text-primary: $palette-white !global;
    $color-text-secondary: $palette-gray-light !global;
    $color-border-default: $palette-white !global;

    // ... другие переопределения ...
}

🔑 Пояснения к практике с темами
1. Семантические переменные ($color-bg-primary)
Это самый важный шаг. Вы никогда не должны использовать $palette-white напрямую в стилях компонентов. Вместо этого вы используете семантическую переменную $color-bg-secondary.
    • На Светлой теме: $color-bg-secondary = $palette-white.
    • На Темной теме: $color-bg-secondary = $palette-gray-dark.
Таким образом, если вы захотите поменять цвет карточек, вы меняете только одну семантическую переменную.
2. Переключение стилей через Миксин/JS
Когда пользователь переключает тему, JS добавляет класс, например, .theme-dark, к элементу body или html.
В вашем главном SCSS-файле (app.scss) вы вызываете миксин темы:
SCSS
// src/app.scss

// 1. Устанавливаем стили по умолчанию (Светлая тема)
body {
    background-color: variables.$color-bg-primary;
    color: variables.$color-text-primary;
}

// 2. ⭐️ Переопределяем стили для Темной темы ⭐️
body.theme-dark {
    // Внутри этого блока мы вызываем миксин,
    // который устанавливает новые значения для семантических переменных.
    @include variables.apply-theme-dark; 
    
    // И теперь все стили используют новые значения переменных
    background-color: variables.$color-bg-primary; 
    color: variables.$color-text-primary;
}

// В компонентах вы всегда используете только семантические переменные:
.card {
    background-color: variables.$color-bg-secondary;
}
3. Использование !global (Внимание!)
В миксине apply-theme-dark я использовал флаг !global.
Если вы используете SCSS-модули (@use) и пытаетесь переопределить переменные внутри миксина, вы должны указать, что это переопределение должно быть глобальным, чтобы оно влияло на переменную, доступную через пространство имен variables.
Этот подход позволяет динамически менять значения семантических переменных при переключении класса темы на body.
