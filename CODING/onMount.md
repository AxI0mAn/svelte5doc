# onMount() - планирует выполнение callBack-функции сразу после монтирования компонента в DOM. 
Функцию необходимо вызывать во время инициализации компонента (но она не обязательно должна находиться внутри компонента; её можно вызвать из внешнего модуля).

Если функция возвращается ( как return ) из метода onMount, она будет вызвана при размонтировании компонента. Использование return позволяет держать код инициализации и код очистки в одном месте. Это называется «командный код». 
return из onMount - это замена для устаревшего onDestroy().

onMount НЕ выполняется внутри компонента, который отображается на сервере.

# Почему мы делаем return в onMount?
Это критически важный момент для предотвращения утечек памяти и багов.
Когда вы пишете return () => ... внутри onMount, вы регистрируете функцию, которая выполнится в момент уничтожения компонента (когда пользователь перешел на другую страницу или скрыл виджет).

Зачем это нужно для setTimeout? Представьте ситуацию:
    1. Компонент появился, сработал onMount, запустился таймер на 500мс.
    2. Через 200мс пользователь быстро нажал кнопку "Назад" и ушел с этой страницы.
    3. Компонент исчез (unmount), но таймер в браузере всё еще тикает.
    4. Через оставшиеся 300мс таймер срабатывает и пытается вызвать функцию animateRange.
    5. Функция animateRange пытается изменить переменную currentValue в компоненте, которого больше нет в памяти.
Результат без return: Ошибка в консоли (попытка обновить уничтоженный компонент) или, что хуже, непредсказуемое поведение интерфейса.

С использованием return:
JavaScript

onMount(() => {
  const timer = setTimeout(animateRange, 500);
  
  // Svelte запомнит эту функцию и вызовет её ПЕРЕД удалением компонента
  return () => {
    clearTimeout(timer); // Мы "убиваем" таймер, если компонент исчез раньше времени
    console.log("Компонент удален, таймер очищен");
  };
});

Это правило касается не только таймеров, но и:
    • Слушателей событий: window.addEventListener.
    • Интервалов: setInterval.
    • Подписок на сокеты или сторонние библиотеки.

Как это применять в дальнейшем (Золотое правило): 
# Чистота кода: Если вы создали что-то "живущее во времени" (таймер, слушатель клика на window), всегда "убирайте за собой" в return внутри onMount.
