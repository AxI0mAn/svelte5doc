4. Основа стилей - файлы с переменными и частями стилей. 
Файлов  может быть несколько и все они должны быть указаны в additionalData файла vite.config.js

_variables.scss	- Содержит только SCSS-переменные. ($color-primary: #123;, $font-size-base: 16px;)
_mixins.scss	- Содержит только общие, многократно используемые миксины. (@mixin flexCenter($gap))
_utils.scss - Содержит только вспомогательные утилитарные классы. (	.u-margin-l, .u-flex-start)
_themes.scss	 - Содержит только логику тем. (@mixin theme-dark, @mixin theme-light)

5. В универсальных компонентах, объявить свои внтрение переменные. 
При использовании коипонента - привязать к внутренним переменным, переменные из этого приложения.

🛡️ Изоляция и чистота компонента (Правильно!) 
🔄 Легкость переиспользования (Правильно!) 
Это идеальный способ адаптации компонента к новому проекту. Вы меняете только строку присвоения, чтобы "подцепить" новую систему именования переменных.

Пример.

есть компонент со своими переменными:

<style lang="scss">
    $link-color: #000;

  button{
      color:  $link-color;
  }
</style> 

Приложение №1. есть variables.scss.  - в котором прописаны стили. Например: $VAR_color: #fff;
в компоненте есть свои переменные, которым присваиваем из variables.scss:

<style lang="scss">
    $link-color: variables.$VAR_color;

  button{
      color:  $link-color;
  }
</style> 

Если мне понадобится этот компонент в другом приложении (Приложение №2. ), где есть СВОЙ файл var.scss.  - в котором прописаны стили. Например: $fontColor: #123456;
то в компоненте мне достаточно заменить переменную: 

<style lang="scss">
    $link-color: var.$fontColor;
  button{
    color:  $link-color;
  }
</style> 

# Не будет конфликта имён переменных стилей, если в _variables.scss есть $color: #123456;  а в компоненте есть своя $color: variables.$color; 
В этом случае вы создаете новую, локальную переменную $color внутри блока < style > компонента. Она существует только в этом компоненте и получает свое значение из глобального модуля variables.

# Хотя конфликта имен не будет, желательно давать разные имена (например, $component-color: variables.$color;) для повышения читаемости и поддерживаемости кода.

7. Переключение темы осуществляем через JS.

JavaScript
function changeSkin(skin) {
  // Удаляем все классы тем
  document.body.classList.remove('theme-dark', 'theme-neon');

  // Если выбрана специфичная тема, добавляем класс
  if (skin === 'dark') {
    document.body.classList.add('theme-dark');
  } else if (skin === 'neon') {
    document.body.classList.add('theme-neon');
  }
  // Для 'default' ничего добавлять не нужно, сработают переменные из :root
}
