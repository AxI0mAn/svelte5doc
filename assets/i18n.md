Ниже представлена подробная инструкция по внедрению i18n (интернационализации) в твой PWA-проект на Svelte 5. Этот алгоритм рассчитан на чистый JavaScript, использование рун и максимальную производительность без сторонних библиотек.
https://gemini.google.com/app/0d3d5a2678c859d8

Инструкция: Реализация i18n в Svelte 5 (RuneMode)
Эта архитектура использует подход «Общее + Страница», что позволяет PWA загружать только 1% данных вместо 100% при открытии конкретной страницы.

1. Файловая структура (Словари)
Храни тексты в папке $lib/i18n/messages. Разделяй их по языкам и страницам.

Plaintext
src/lib/i18n/messages/
├── ru/
│   ├── common.json       // Кнопки, меню, футер
│   ├── home.json         // Текст главной
│   └── basic-math.json   // Статья "Базовая математика"
├── en/
│   ├── common.json
│   └── ...
Пример common.json:

JSON
{
  "buttons": { "save": "Сохранить", "back": "Назад" },
  "nav": { "settings": "Настройки" }
}


2. Роутинг с параметром языка
Чтобы пути были SEO-дружелюбными и работали в PWA, используй опциональный параметр [[lang]].

Новая структура путей:

src/routes/[[lang]]/+layout.svelte — главный макет.

src/routes/[[lang]]/+page.svelte — главная страница.

src/routes/[[lang]]/(math)/basic/+page.svelte — вложенная страница.


3. Логика "Переводчика" (src/lib/i18n.svelte.js)
Создай файл, который будет динамически подгружать нужный JSON и предоставлять его компонентам через руны.

JavaScript
import { page } from '$app/state';

// Реактивные переменные для хранения текстов
let uiTexts = $state({}); 
let pageTexts = $state({});

// Главная руна, которая объединяет общие и страничные тексты
export const t = $derived.by(() => {
    return { ...uiTexts, ...pageTexts };
});

// Функция загрузки нужного файла
export async function loadTranslations(lang, routeId) {
    try {
        // 1. Загружаем общие фразы (common)
        const common = await import(`./messages/${lang}/common.json`);
        uiTexts = common.default;

        // 2. Определяем имя файла страницы из пути (routeId)
        // Если путь /ru/basic-math, берем "basic-math"
        const segments = routeId.split('/');
        const pageName = segments[segments.length - 1] || 'home';
        
        const specific = await import(`./messages/${lang}/${pageName}.json`);
        pageTexts = specific.default;
    } catch (e) {
        console.warn('Translation file not found', e);
        pageTexts = {}; 
    }
}


4. Подключение в макете (+layout.svelte)
Используй $effect, чтобы реагировать на смену URL и подгружать нужные словари автоматически.

Svelte
<script>
  import { page } from '$app/state';
  import { loadTranslations } from '$lib/i18n.svelte.js';

  // Следим за языком в URL и ID страницы
  $effect(() => {
    const lang = page.params.lang || 'ru'; // 'ru' - fallback по умолчанию
    loadTranslations(lang, page.route.id);
  });
</script>

<html lang={page.params.lang || 'ru'}>
  <slot />
</html>


5. Работа с ссылками (Навигация)
Чтобы при переходе не "терять" выбранный язык, ссылки должны учитывать текущий префикс.

В компоненте навигации:

Svelte
<script>
  import { page } from '$app/state';
  // Если язык в URL есть, добавляем его в начало ссылок
  let lp = $derived(page.params.lang ? `/${page.params.lang}` : '');
</script>

<a href="{lp}/basic">Базовая математика</a>
<a href="{lp}/settings">Настройки</a>


6. Переключатель языка (Настройки)
Для смены языка используй функцию goto, которая изменит URL, что в свою очередь активирует $effect и обновит все тексты.

JavaScript
import { goto } from '$app/navigation';
import { page } from '$app/state';

function setLanguage(newLang) {
    // Сохраняем выбор в localStorage (для следующего визита)
    localStorage.setItem('pref_lang', newLang);

    // Меняем префикс в текущем URL
    const currentPath = page.url.pathname;
    const langRegex = /^\/(ru|en|es|de|fr)/;
    
    let newPath;
    if (langRegex.test(currentPath)) {
        newPath = currentPath.replace(langRegex, `/${newLang}`);
    } else {
        newPath = `/${newLang}${currentPath}`;
    }

    goto(newPath);
}


7. Алгоритм определения языка при первом входе
В твоем PWA логика должна быть следующей:

Проверка URL: Если в ссылке уже есть /en/, используем его.

Проверка LocalStorage: Если в ссылке нет языка, смотрим, что сохранено в настройках.

Язык браузера: Если настроек нет, берем navigator.language.

Fallback: Если ничего не подошло — ставим русский.

Главные преимущества этой системы:
Производительность: Страница "Engineer" никогда не загрузит текст со страницы "Basic".

Чистота: Ты используешь только возможности Svelte 5 (руны $state и $derived).

PWA-ready: Все импортированные JSON кэшируются браузером как отдельные файлы, экономя память устройства.
