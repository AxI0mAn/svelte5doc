# ГОТОВЫЙ ПРОВЕРЕННЫЙ КОД В a1_step-by-step.docx





В вашем случае, поскольку вы не используете TypeScript, но хотите обеспечить правильное разрешение путей (алиасов) и устранить ошибки, связанные с языковым сервером, я рекомендую минимальный и безопасный набор настроек.
Этот код наследует все необходимые настройки от внутреннего файла SvelteKit и просто включает проверку для ваших файлов JavaScript (JS/Svelte) :
tsconfig.json (в корне проекта)
JSON
{
  // Наследует все базовые настройки от внутреннего файла SvelteKit
  "extends": "./.svelte-kit/tsconfig.json",
  
  // Компилятор TypeScript
  "compilerOptions": {
    // Разрешить использовать JS-файлы
    "allowJs": true,
    
    // Включить проверку типов в JS-файлах (то есть Svelte-файлах)
    // Это обеспечивает автодополнение и помогает устранять ошибки
    "checkJs": true
  },
  
  // Убедиться, что он ищет файлы в папке src
  "include": ["src/**/*.d.ts", "src/**/*.js", "src/**/*.svelte"]
}
Почему это решает ваши проблемы:
    • Устраняет ошибку "File not found": Четкое указание include помогает TypeScript правильно проиндексировать все ваши .svelte и .js файлы, устраняя ошибки, связанные с поиском старых путей.
    • Решает проблемы с алиасами: Наследуя настройки через "extends", ваш редактор начинает правильно понимать все ваши алиасы ($lib, $components).
    • Синхронизирует Runes Mode: Правильная настройка компилятора помогает SvelteKit применить все глобальные настройки (включая runes: true) ко всем компонентам без сбоев.
