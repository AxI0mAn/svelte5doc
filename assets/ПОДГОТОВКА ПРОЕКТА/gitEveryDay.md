📘 Шпаргалка Git для ежедневной работы

# 🛠 Базовый цикл (Новая задача)

  Обновить основной код:
    git checkout main — перейти в главную ветку.
    git pull origin main — забрать изменения из облака.

  Создать ветку под задачу:
    git checkout -b feat-name — создаёт новую ветку и сразу переходит в неё.

  Сохранить работу:
    git add . — подготовить все изменения.
    git commit -m "feat: описание" — создать коммит.

  Отправить в облако:
    git push origin feat-name — создать ветку на GitHub и закинуть код.

  🔍 Просмотр и навигация
    Статус: git status — что изменено, что готово к коммиту.

  Список веток: git branch -a — показывает локальные и удаленные (remote) ветки.

  Переход: git checkout name — перейти в существующую ветку (исправлено).

  Граф в терминале: git log --oneline --graph --all — если лень смотреть в плагин VSCodium.

  Если вы хотите вывести граф так, чтобы он не "замораживал" терминал, а просто напечатался и оставил вас в командной строке, добавьте --no-pager перед командой:
    git --no-pager log --oneline --graph --all -n 10 (покажет только последние 10 записей).

# 🆘 На всякий случай (Исправление ошибок)

Отмена коммита (мягкая): git reset --soft HEAD~1 — коммит исчезнет, но весь ваш код останется в файлах (удобно, если забыли что-то добавить или ошиблись в описании).

Отмена коммита (жесткая): git reset --hard HEAD~1 — ВНИМАНИЕ: удаляет коммит вместе со всеми изменениями в коде. Возвращает состояние к предыдущему коммиту.

Удалить ветку (локально): git branch -d name — удалить ветку после того, как она слита в main. git branch -d feat-pwa

Исправить опечатку в последнем коммите: git commit --amend -m "новое описание" — если еще не делали push.

# 🧪 Специфическое для вашего проекта (Svelte + GH Pages)
Синхронизация типов: npx svelte-kit sync — если VSCodium «сошел с ума» и видит ошибки в путях, которых нет.

Локальная проверка сборки: npm run build && npm run preview — проверить, работает ли роутинг и редирект на /en/ до того, как отправлять код на GitHub.
