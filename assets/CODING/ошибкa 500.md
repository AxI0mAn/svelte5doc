Vite/SvelteKit кэширует старую версию файла!!!

🛠️ Как исправить ошибку 500

Самый быстрый и надежный способ решить эту проблему — сбросить кэш сборщика и запустить чистую установку.

1. 🧹 Очистка кэша SvelteKit/Vite (Обязательный шаг)
Vite и SvelteKit кэшируют сгенерированный код, и иногда они не видят, что вы переключили режим компиляции (Legacy -> Runes).
Выполните следующие команды в терминале, чтобы удалить все временные файлы, сгенерированные сборщиком:

Bash

# Удалить кэш Vite
rm -rf .vite

# Удалить сгенерированные файлы SvelteKit

rm -rf .svelte-kit


2. ⚡️ Перезапуск сервера
После очистки кэша обязательно полностью остановите сервер (Ctrl+C), а затем запустите его снова:
Bash
npm run dev
Это заставит Vite и SvelteKit скомпилировать все файлы с нуля, применив глобальную настройку runes: true к макету src/routes/(home_all)/+layout.svelte.
