# 1. 🖼️ Использование SVG как <img> (URL)
# 2. 🧩 Использование SVG как <svg> (Svelte-компонент)

1. 🖼️ Использование SVG как <img> (URL)
Для использования SVG как обычного изображения, которое загружается по URL-адресу, вы используете стандартный механизм Vite.
✅ Как это работает:
    1. Импорт: Вы используете импорт по умолчанию (без суффикса):
       JavaScript
       import iconUrl from '$lib/assets/my-icon.svg';
    2. Vite: Vite обрабатывает этот импорт как статический ассет. Он копирует SVG в папку сборки (build/assets/) и возвращает публичный URL-адрес к этому файлу.
    3. Использование: В шаблоне вы используете переменную как обычный src.
       HTML
       <img src={iconUrl} alt="Описание иконки" />
Вывод: Этот способ не требует дополнительных настроек, так как он поддерживается вашей базовой конфигурацией Vite. Это идеально подходит, когда вам не нужно менять цвет или другие стили SVG через CSS родительского компонента.

2. 🧩 Использование SVG как <svg> (Svelte-компонент)
Для встраивания SVG непосредственно в DOM (inline <svg>...</svg>), чтобы вы могли легко стилизовать его с помощью CSS (например, менять fill или stroke), вы используете специальный суффикс, предоставленный плагином SvelteKit.
✅ Как это работает:
    1. Импорт: Вы используете специальный суффикс ?svelte:
       JavaScript
       import IconComponent from '$lib/assets/my-icon.svg?svelte';
    2. SvelteKit: Плагин SvelteKit распознает суффикс ?svelte и преобразует SVG-файл в обычный Svelte-компонент, который экспортирует SVG-разметку.
    3. Использование: Вы используете импортированное имя как Svelte-компонент:
       HTML
       <IconComponent class="main-icon" />
       
       <style>
         .main-icon {
           color: var(--primary-color); /* Используется для fill/stroke, если они не заданы явно в SVG */
         }
       </style>
Вывод: Этот способ также не требует дополнительных настроек. Он встроен в плагин @sveltejs/vite-plugin-svelte, который вы уже используете.
