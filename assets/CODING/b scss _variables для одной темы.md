📝 Пример файла _variables.scss (Лучшие практики)
В соответствии с лучшими практиками SCSS, файл _variables.scss должен быть узкоспециализированным и содержать только переменные для настройки глобальных параметров проекта. Эти переменные следует группировать по категориям (Цвета, Типографика, Макет) для удобства поддержки.
Вот пример кода и пояснения:
SCSS
// =====================================================================
// _variables.scss
// Глобальные SCSS-переменные проекта, сгруппированные по категориям.
// =====================================================================

// ---------------------------------------------------------------------
// 1. ЦВЕТА БАЗОВОЙ ПАЛИТРЫ (Raw Colors)
// Рекомендуется называть их по назначению, а не по оттенку (e.g., $color-primary)
// ---------------------------------------------------------------------

$color-white: #ffffff;
$color-black: #000000;

// -- Основные цвета бренда
$color-primary-base: #2a7fec;       // Базовый синий
$color-primary-dark: #1e67c8;
$color-primary-light: #5ba3f2;

// -- Семантические цвета (для уведомлений/статусов)
$color-success: #28a745;
$color-danger: #dc3545;
$color-warning: #ffc107;

// -- Цвета фона и текста
$color-text-dark: $color-black;     // Используется для основного текста
$color-text-light: $color-white;    // Используется для текста на темном фоне
$color-background: #f8f8f8;         // Светлый фон страницы
$color-border: #cccccc;             // Цвет границы по умолчанию

// ---------------------------------------------------------------------
// 2. ТЕМЫ (Game Theme)
// Переменные для конкретной темы (например, $bg_game из вашего проекта)
// ---------------------------------------------------------------------

$bg_game: #0f1c2c;                  // Темный игровой фон
$game-accent: #ff4500;              // Акцентный оранжевый

// ---------------------------------------------------------------------
// 3. ТИПОГРАФИКА
// ---------------------------------------------------------------------

// -- Шрифты
$font-stack-base: 'Roboto', sans-serif;
$font-stack-heading: 'Georgia', serif;

// -- Размеры
$font-size-base: 1rem;              // Базовый размер (16px)
$font-size-small: 0.875rem;
$font-size-large: 1.25rem;
$font-size-h1: 2.5rem;
$font-size-h2: 2rem;

// -- Веса
$font-weight-regular: 400;
$font-weight-bold: 700;

// ---------------------------------------------------------------------
// 4. МАКЕТ И РАЗМЕРЫ (Layout & Spacing)
// ---------------------------------------------------------------------

// -- Интервалы (используются для margin/padding)
$spacing-unit: 8px;                 // Базовая единица (для ритма)
$spacing-xs: $spacing-unit * 0.5;   // 4px
$spacing-sm: $spacing-unit;         // 8px
$spacing-md: $spacing-unit * 2;     // 16px
$spacing-lg: $spacing-unit * 4;     // 32px

// -- Размеры экрана (Breakpoints)
$breakpoint-sm: 576px;
$breakpoint-md: 768px;
$breakpoint-lg: 992px;
$breakpoint-xl: 1200px;

🔑 Пояснения к лучшим практикам
1. Группировка
Переменные разделены на логические блоки (ЦВЕТА, ТИПОГРАФИКА, МАКЕТ). Это позволяет легко найти или изменить нужный параметр.
2. Принцип "Один Источник Истины" (Single Source of Truth)
В этом файле все числа и цвета задаются один раз. Другие переменные (например, $color-text-dark) ссылаются на базовые ($color-black). Это гарантирует, что если вы решите изменить базовый черный, все зависимые переменные изменятся автоматически.
3. Использование rem
Для размеров шрифтов ($font-size-*) и часто для интервалов рекомендуется использовать единицы rem (root em), а не px. Это обеспечивает лучшую доступность (Accessibility), так как размеры масштабируются в зависимости от настроек браузера пользователя.
4. Единица Сетки ($spacing-unit)
Использование базовой единицы ($spacing-unit: 8px;) для интервалов (как в секции "МАКЕТ") является лучшей практикой. Это позволяет создавать единый вертикальный/горизонтальный ритм (на основе 8-точечной сетки), где все отступы являются множителями этой базовой единицы.
5. Именование
Используется префикс $ (обязательно для SCSS) и понятное, структурированное именование (например, $color-primary-base или $spacing-md).
