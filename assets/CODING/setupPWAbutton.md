Реализация кнопки «Установить» в PWA не так проста, как обычная ссылка на скачивание. Браузер сам решает, когда приложение «достойно» установки, но он предоставляет нам событие, которое мы можем перехватить.

Вот пошаговый алгоритм, как это сделать:

1. Как это работает (Теория)
Браузер генерирует событие beforeinstallprompt. Нам нужно:

Перехватить это событие и сохранить его в переменную.

Показать пользователю вашу кастомную кнопку «Установить» (по умолчанию она должна быть скрыта).

При нажатии на кнопку вызвать метод .prompt() из сохраненного события.

2. Реализация кода
Вам нужно создать логику, которая будет слушать браузер. Вот пример реализации, который можно вынести в отдельный файл или использовать внутри компонента.

Шаг А: Скрипт перехвата
JavaScript
let deferredPrompt; // Переменная для хранения события

window.addEventListener('beforeinstallprompt', (e) => {
  // Предотвращаем автоматическое появление системного баннера
  e.preventDefault();
  // Сохраняем событие, чтобы вызвать его позже
  deferredPrompt = e;
  
  // Здесь нужно показать вашу кнопку установки
  // Например, убрать класс 'hidden' или поменять состояние в Svelte/React
  const installBtn = document.getElementById('install-button');
  if (installBtn) {
    installBtn.style.display = 'block';
  }
});
Шаг Б: Логика нажатия на кнопку
JavaScript
const installButton = document.getElementById('install-button');

installButton.addEventListener('click', async () => {
  if (!deferredPrompt) return;

  // Показываем системное окно установки
  deferredPrompt.prompt();

  // Ждем ответа пользователя
  const { outcome } = await deferredPrompt.userChoice;
  console.log(`Результат установки: ${outcome}`);

  // Очищаем переменную, событие можно использовать только один раз
  deferredPrompt = null;

  // Скрываем кнопку снова
  installButton.style.display = 'none';
});
Шаг В: Обработка успешной установки
JavaScript
window.addEventListener('appinstalled', () => {
  console.log('PWA успешно установлено!');
  // Очищаем интерфейс
  deferredPrompt = null;
});

3. Обязательные условия (Checklist)
Кнопка не появится, если PWA не соответствует критериям Google:

Manifest.json: Должен быть корректно заполнен (иконки, имя, start_url, display: standalone).

Service Worker: Должен быть зарегистрирован и работать (хотя бы с пустым обработчиком fetch).

HTTPS: Сайт должен работать по защищенному протоколу (исключение — localhost).

Повторная установка: Если приложение уже установлено, событие beforeinstallprompt не сработает.

4. Нюансы для разных платформ
Android (Chrome): Работает идеально по схеме выше.

iOS (Safari): Плохие новости. Safari не поддерживает событие beforeinstallprompt. Пользователи iPhone могут установить PWA только вручную через кнопку «Поделиться» (Share) -> «На экран "Домой"» (Add to Home Screen).

Совет: Для iOS обычно показывают текстовую подсказку с картинкой (инструкцию), как именно нажать кнопку «Поделиться».

Desktop (Chrome/Edge): Работает так же, как на Android. В адресной строке также появится иконка установки.

Резюме для вашего проекта
Поскольку вы используете Svelte, вы можете создать стор (store) или переменную состояния canInstall, которая будет становиться true внутри обработчика beforeinstallprompt.

Svelte
{#if canInstall}
  <button on:click={handleInstallClick}>Установить приложение</button>
{/if}
Это будет выглядеть профессионально и нативно!
