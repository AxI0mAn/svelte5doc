# Если при попытке запуска из терминала - получаем ТАКУЮ ошибку:

sh-5.2$ npm run dev -- --open
env: ‘node’: Нет такого файла или каталога



# Тогда нужно применить решение Проблемы node в VSCodium.

1. Определите вашу основную оболочку: В вашем основном терминале выполните:

Bash

echo $SHELL

Вероятно, вы увидите /bin/bash или /bin/zsh.

2. Настройте VSCodium:

# применив этот шаг - новые проекты создаются без оболочки sh-2.5 !!!

Откройте настройки VSCodium (File > Preferences > Settings).

В поиске введите: "terminal integrated default profile" (или "терминал интегрированный профиль по умолчанию").

Выберите профиль, который соответствует вашей основной оболочке (например, Bash или Zsh).

3. Перезапустите VSCodium.

4. Откройте новый терминал VSCodium. Теперь приглашение должно выглядеть иначе (например, daxio@daxio-iMac:~/...$), и команды node -v и npm run dev должны работать без полного пути.

5. Когда приглашение сменилось - это означает, что настройка терминала VSCodium сработала! 🎉 Теперь терминал загружает ваши конфигурационные файлы.

Однако он загрузил старую версию Node.js (v16.20.2). Это происходит потому, что в вашем .bashrc или .zshrc либо не прописана команда nvm use для выбора версии, либо вы не указали, что хотите использовать Node 22 по умолчанию.

Мы можем легко это исправить, используя nvm прямо в этом же терминале VSCodium.

6. 🛠️ Урок 0.5: Установка и Фиксация Node.js v22
📝 Шаг 1. Проверяем, Установлена ли v22
Возможно, вы ее уже устанавливали, но просто не выбрали. В новом терминале VSCodium выполните:

Bash

nvm ls

Если вы увидите v22.x.x в списке, переходите к Шагу 3.

Если нет, переходите к Шагу 2.

📝 Шаг 2. Установка Node.js v22 (Если не Установлена)
Установите последнюю версию.

Bash

nvm install 22

📝 Шаг 3. Активация v22 для Текущей Сессии
Активируйте ее в текущем терминале VSCodium:

Bash

nvm use 22

📝 Шаг 4. Установка Версии по Умолчанию (Символическая Ссылка)
Чтобы каждый раз при открытии нового терминала VSCodium автоматически использовалась v22, мы устанавливаем ее как версию по умолчанию (default).

Bash

nvm alias default 22
Теперь, когда VSCodium загрузит терминал, он будет использовать эту версию.

📝 Шаг 5. Проверка и Запуск
Проверьте версию в текущем терминале:

Bash

node -v
Ожидаемый результат: v22.x.x.

Теперь запустите проект, используя простую команду (без полного пути, так как node теперь виден):

Bash

npm run dev -- --open

Всё работает и в браузере открылся проект!
