🚀 Алгоритм подключения проекта к GitHub

Шаг 1: Инициализация проекта на ПК
    1. Создайте локальный репозиторий: Откройте терминал в корневой папке вашего проекта (где находится package.json): 
       git init
    2. Проверьте статус: 
       git status 
       (Вы увидите список всех файлов как "untracked").
Шаг 2: Создание и настройка .gitignore (Критически важно!)
Файл .gitignore говорит Git, какие файлы и папки нужно игнорировать и не включать в репозиторий. Это предотвращает загрузку огромных, временных, или секретных файлов.
    1. Создайте файл: В корневой папке проекта создайте файл с именем .gitignore.
    2. Базовое содержимое для SvelteKit/Node.js: Вам необходимо исключить все временные папки, кэш и папку сборки.

       //  .gitignore

# ⚙️ Зависимости и сгенерированные файлы Node/npm
/node_modules
npm-debug.log*
yarn-debug.log*
pnpm-debug.log*

# 🛠️ Сгенерированные папки SvelteKit и Vite (КРИТИЧНО)
# Они должны быть исключены, так как генерируются командой 'npm run build' или 'npm run dev'.
.output
.vercel
.netlify
.wrangler
/.svelte-kit
/build 
/.vite

# 💻 Файлы среды разработки и кэш
.DS_Store
Thumbs.db

# 🔑 Переменные окружения (Обязательно, чтобы не загрузить секреты)
.env
.env.local
.env.*
!.env.example

# 🧰 Файлы редактора (VSCodium/VS Code)
.vscode/

# ⏱️ Временные файлы Vite
*.timestamp-*

Шаг 3: Первый коммит
    1. Добавьте все файлы:
       Bash
       git add .
       Нюанс: Git добавит все, кроме файлов, указанных в .gitignore.
    2. Сделайте коммит (фиксацию изменений):
       Bash
       git commit -m "Initial commit of SvelteKit project with Git Pages config"

Шаг 4: Подключение к GitHub
    1. Создайте репозиторий на GitHub: Перейдите на GitHub, нажмите New repository и создайте пустой репозиторий (без Readme, без .gitignore).
    2. Свяжите локальный и удаленный репозитории: Вернитесь в терминал и выполните команды, которые вам покажет GitHub (обычно это):
       Bash
       git remote add origin https://github.com/ВАШ_USERNAME/ИМЯ_РЕПОЗИТОРИЯ.git
       git branch -M main
    3. Отправьте код (push):
       Bash
       git push -u origin main
       Ваш код (за исключением игнорируемых файлов) теперь находится на GitHub.

Нюансы и хитрости .gitignore

1. ⚠️ Локальные переменные окружения
Если вы используете переменные окружения для секретных ключей или настроек, которые должны отличаться от продакшена:
    • Хитрость: Используйте файл .env.local для локальных ключей и обязательно добавьте его в .gitignore.
    • Правило: Никогда не коммитьте секретные ключи или токены в публичный репозиторий.
2. Папки сборки SvelteKit
Обязательно включите в .gitignore все временные папки:
    • /build: Результат работы адаптера (adapter-static). Это финальная папка для публикации на GH Pages. Она всегда должна быть в .gitignore, так как она генерируется при сборке.
    • /.svelte-kit: Кэш SvelteKit, который постоянно меняется.
    • /.vite: Кэш Vite.
3. Файлы SvelteKit (JS-проект)
Если бы вы использовали TypeScript, SvelteKit генерировал бы файл с типами. Поскольку у вас чистый JS, вам нужно исключить только файлы кэша.
Главный принцип: В Git должен находиться только исходный код, который необходим для воспроизведения проекта на другом компьютере. Все, что генерируется командами (npm install, npm run build), должно быть проигнорировано.
