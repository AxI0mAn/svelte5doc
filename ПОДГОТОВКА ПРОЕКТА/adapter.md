# Зачем нужна установка @sveltejs/adapter-static и настройка svelte.config.js и почему "@sveltejs/adapter-auto" не подходит?



Адаптеры в SvelteKit, такие как @sveltejs/adapter-static и @sveltejs/adapter-auto, определяют, как ваш скомпилированный код (HTML, CSS, JS) будет развернут (deployed) и работать в "боевых" условиях.
Роль Адаптеров в SvelteKit. 

Адаптер — это плагин, который берет готовый билд вашего SvelteKit-проекта (созданный командой npm run build) и преобразует его в формат, подходящий для конкретной хостинг-среды.


1. @sveltejs/adapter-auto (По умолчанию)
Этот адаптер предназначен для гибкости и автоматического определения среды.

- Предполагает серверный рендеринг (SSR) или генерацию статических страниц (SSG). Хостинги, поддерживающие Node.js-сервер или Serverless Functions (Vercel, Netlify, Cloudflare Pages/Workers).
 
- Генерирует код, который должен быть запущен на сервере (или в бессерверной функции), чтобы отвечать на запросы (рендерить страницы на лету).
Полноценные приложения с API, базами данных и динамическим контентом.

Почему не подходит для вас:
Он не гарантирует создание чистого SPA и может потребовать специфической настройки сервера, даже если вы не используете серверные функции. Он не генерирует простую папку с файлами, которые можно просто "бросить" на любой статический хостинг.



2. @sveltejs/adapter-static (Ваш выбор)
Этот адаптер предназначен специально для создания чистых статических сайтов и SPA.

- Создает только статические файлы (HTML, CSS, JS, изображения). Отключает SSR и все серверные функции SvelteKit (вроде +server.js). SPA (как в вашем случае), обычные статические сайты, блоги, документация.

- Генерирует папку build (или dist) с чистыми файлами, которые можно развернуть на любом статическом веб-сервере (GitHub Pages, S3, Nginx, Apache).

Почему подходит для вас
Вы планируете SPA с LocalStorage, который не требует сервера для работы. Этот адаптер идеально подходит для генерации минимального, быстрого и переносимого кода.
🔑 Ключевые настройки в svelte.config.js для SPA
Чтобы adapter-static работал как чистое SPA, необходимо явно указать две вещи:
1. adapter: adapter({...})
Установка самого адаптера.
2. fallback: '200.html'
Это самая важная настройка для SPA-роутинга:
    • В обычном статическом сайте, если пользователь запрашивает /about, сервер ищет файл /about/index.html.
    • В SPA с клиентским роутингом, пользователь может ввести /game-stats напрямую. Сервер не найдет файл /game-stats/index.html.
    • Настройка fallback: '200.html' говорит статическому серверу: "Если ты не нашел запрошенный файл, верни главный файл приложения, обычно 200.html (или index.html)."
    • После этого главный файл (который содержит ваш Svelte-код) загрузится, и уже клиентский роутер SvelteKit разберется с путем /game-stats и покажет нужный компонент.
Резюме
Вам нужна установка @sveltejs/adapter-static и настройка fallback: '200.html' в svelte.config.js, чтобы гарантировать, что ваш проект будет скомпилирован в переносимую, статическую SPA-директорию, которая может быть развернута на любом статическом хостинге без необходимости в серверном Node.js-окружении.

1. Как Установить сам адаптер adapter-static?
2. Где нужно указать fallback: '200.html' ?

3. Что нужно для работы с SCSS?


1. Как установить адаптер? 🛠️

Используйте следующую команду в терминале в корне вашего проекта: 
npm install -D @sveltejs/adapter-static
-D или --save-dev указывает, что это зависимость разработки (devDependencies), поскольку адаптер нужен только для сборки проекта, а не для его работы в браузере.

2. Где указать fallback: '200.html'? ⚙️
Эту настройку нужно указать в конфигурационном файле SvelteKit: svelte.config.js.

Поскольку, оригинальный проект использует SCSS, то обязательно нужно добавить vitePreprocess и установить сам препроцессор.

3. Установка SCSS:
Используйте следующую команду в терминале в корне вашего проекта: 
npm install -D sass

4. Добавление vitePreprocess и adapter-static в svelte.config.js:
JavaScript
// svelte.config.js

import adapter from '@sveltejs/adapter-static'; // Импортируем адаптер 

import { vitePreprocess } from '@sveltejs/vite-plugin-svelte'; // <-- Импортируем

/** @type {import('@sveltejs/kit').Config} */
const config = {
    // Добавляем секцию preprocess и используем vitePreprocess()
    preprocess: vitePreprocess(), // <-- Используем

    kit: {
        adapter: adapter({
            fallback: '200.html'
        })
    }
};

export default config;
