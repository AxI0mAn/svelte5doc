# 🚀 vite.config.js: Конфигурация Сборщика Проекта

Этот файл отвечает за то, как Vite (сборщик) обрабатывает весь остальной код (JavaScript, TypeScript, CSS, картинки) и управляет сервером разработки.

Базовый минимум функциональности:
-------------------------------------------------
server.port / server.host	- Настройка сервера разработки. Например, чтобы запустить сервер не на localhost:5173, а на другом порту или на всех сетевых интерфейсах.	
    Полезно для совместной работы в локальной сети или при работе с Docker.

-------------------------------------------------
resolve.alias	- Создание алиасов (псевдонимов) для импорта, которые используются Vite (для JS, CSS и т.д.). (Связан с kit.alias в svelte.config.js).	
    Позволяет создавать чистые пути импорта.

-------------------------------------------------
plugins	- Подключение сторонних плагинов Vite. Например, плагины для оптимизации изображений, интеграции с другими фреймворками или специальной обработки CSS.	
    Критически важно для расширения функционала сборки.

-------------------------------------------------
css	- Настройка CSS/Sass/Less — например, автоматическое добавление префиксов браузеров или глобальные переменные (CSS Modules).	
    Полезно для работы с Sass/Less.

# 🤝 Взаимодействие файлов
Важно понимать, что SvelteKit работает поверх Vite:

Когда вы запускаете проект, Vite запускается первым, используя свою конфигурацию (vite.config.js).

Vite подключает плагин SvelteKit, который, в свою очередь, считывает настройки из svelte.config.js.

Таким образом, vite.config.js управляет общим процессом сборки и инструментами, а svelte.config.js — тем, как Svelte обрабатывает специфические .svelte файлы.


====================================

# Почему не нужно импортировать sass в vite.config.js?
Vite очень умный: Когда вы используете SCSS в файле Svelte (<style lang="scss">) или импортируете SCSS-файл в JavaScript (import './style.scss';), Vite автоматически обнаруживает, что вам нужен препроцессор.

Если sass установлен, Vite автоматически использует его для обработки SCSS без необходимости его явного импорта и настройки в vite.config.js.

# Настройка Алиасов (resolve.alias)

Если вы хотите использовать короткий путь $utils/ вместо длинного src/lib/components/utils/, вам нужно настроить секцию resolve.alias в vite.config.js.

vite.config.js

JavaScript

import { sveltekit } from '@sveltejs/kit/vite';
import { defineConfig } from 'vite';
import path from 'path'; // ⭐️ Импортируем модуль path для корректной работы с путями

export default defineConfig({
  plugins: [sveltekit()],
  
  resolve: {
    alias: {
      // ⭐️ Настройка вашего нового алиаса:
      // __dirname - это путь к текущей папке проекта
      '$utils': path.resolve('./src/lib/components/utils'), 
    }
  },
});
Важно: Для надежной работы с путями в Node.js, используйте модуль path (он встроен и не требует установки) и метод path.resolve().

# Удаление комментариев в Продакшене (Спецобработка)

Vite по умолчанию удаляет большинство комментариев при сборке продакшен-версии (минификация CSS и JS). Вам не нужно добавлять специальный плагин для этой базовой функции.

Минификация (удаление комментариев) происходит благодаря плагину terser (для JS) и встроенному минификатору CSS, которые работают автоматически при npm run build.

Если вы хотите быть абсолютно уверены, что все комментарии удалены, убедитесь, что в ваших package.json нет настроек, отключающих минификацию в режиме продакшена.
